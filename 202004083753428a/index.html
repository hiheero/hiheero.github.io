    技术分享 | 安卓视图绑定工具 ViewBinding
    2020/04/08 技术 技术分享 安卓

    从此告别 findViewById 和 Butter Knife。

    前言

    在安卓开发过程中我们总是需要获取 XML 布局中的 View ID，以此对 UI 组件进行操控。早期我们只能使用 findViewById 这个 API，并且代码比较繁琐。2013 年左右安卓大神 Jake Wharton 开源了框架 Butter Knife，可以使用 Bind(“viewId”) 的方式来获取 View ID。这两年谷歌支持 Kotlin 为安卓开发主流语言，我们开始使用 Android Kotlin extensions，在文件中导入布局文件就可以使用 View ID，不用其他额外操作，非常方便。

    目前，谷歌在 Android Studio 3.6 Canary 11 及更高版本中加入了新的视图绑定工具 ViewBinding。所以，为了使用 ViewBinding，必须将 Android Studio 升级至 3.6 版本或以上。

    总结

    ViewBinding 相较于 findViewById 和 Butter Knife 两种方式相比无疑方便了许多，ViewBinding 还可以避免类型转换和空指针异常的问题。和 Android Kotlin extensions 相比较都差不多，说不上谁更好。

    目前 Jake Wharton 在 Butter Knife 开源库中添加了如下一句话：

    Attention: Development on this tool is winding down. Please consider switching to view binding in the coming months.

    想必 ViewBinding 在未来的地位和作用将不言而喻了吧。Butter Knife 也终成为时代的眼泪。
